# Agent Mandates

- **Think More, Act Less:** Do not rush into implementation. Deeply analyze the game context (TFT) and user requirements before writing code.
- **In-Game Assistant Context:** Always remember the tool is used *during* a live match. Speed, clarity, and ease of use (one-handed operation, large hit targets) are paramount.
- **Default State:** A game starts at Level 1. The tool must reflect the current state of a match, not an end-game state.
- **UI Efficiency:** Minimize large decorative headings. Maximize space for functional elements (Board, Pool, Controls).
- **Confirmation Protocol:** For non-trivial changes, describe the plan in detail and wait for an explicit "Go" or "Build it."
